Here’s a tiny collection of fun, usually unfinished, small weekend toy-projects/hacks. Doodles!

Simple, but Useful

Tiny tools I’ve actually used on a regular basis at different times:

  • runonchange: just a super bloated inotifywait loop (10x-bloat version of the shell script); mostly used to play with golang. would write it very differently today. Surprisingly Just Works so often that I keep using it. Notes from a lightning talk I gave on this.

  • punch: simple time keeper; used in my early days as an engineer to learn about getopts builtin, and then again years later to learn about golang. Also surprised me when I ended up actually being able to use and trust it for a couple years.

  • tiny scripts for poking about a machine: pagebin and descbin

  • fpreviewext: tiny, simple project to display file://…​; contents inlined - used mostly to learn how to write a chrome extension with TypeScript, but now I really use this tool daily.

  • ~/.host/pick script: back in 2009-ish I was trying to share my dotfiles between machines, but wanted slight tweaks depending on certain attributes of the host machine said files were downloaded to. This solution has worked with nearly zero changes since then, surprisingly well. It’s just a file-naming convention really.

Abandonned, but Fun

  • Playing with golang as part of a Real Analysis exam: a Riemann sum calculator.

  • webdeskgo: playing with LXC and a tiny golang web server to generate pdf booklets by shelling out to pdfbook and not worry too much about what gets uploaded. I’d highly discourage using this an any exposed environment.

    • Actually, I’d discourage using this at all. It’s way easier to just call pdfbook forever, and never spend the effort installing this thing. This was more an attempt to prototype what lxc can be used for, and how I might write tiny webapps I don’t care about the security of.

  • scanfix; tiny golang [1] tool I was using for a while when I’d scan textbooks on a flatbed scanner.

Some partially completed doodles involving PC backups:

  • sysrestic: spirit of convention by configuration; never-quite-completed attempt to capture all the annoying startup/setup work to have a sane, encrypted backup system that works with a modern (systemd) PC.

  • friends backup friends: mostly design documents and the research/starting-stages to prototype a fun concept. Ran out of weekend-energy on this one, but might go back to it some day. Was fun to think through.

I’m mostly keeping this page here for my own easy reference, but if you find something useful let me know, I’d love to hear it!

1. If it’s not evident yet from this page, I try very to avoid writing long scripts in Bash, and write them in a proper programming language instead. Hence all the strange golang tools.
